# Gmail Reply Templates

목표: 실제 문의가 들어왔을 때 Gmail 답장을 빠르게 작성하고, 범위와 가격을 일관되게 안내한다.

## 1. 샘플 파일 요청

상태 전환:

```text
NEW -> NEEDS_SAMPLE
```

제목:

```text
Re: 엑셀/CSV 자동화 맞춤 제작 문의
```

본문:

```text
안녕하세요.
문의 감사합니다.

작업 가능 범위와 견적을 확인하려면 아래 항목이 필요합니다.

1. 현재 사용 중인 파일 형식: CSV / XLSX / 구글시트 / 기타
2. 반복해서 하는 작업: 예) 중복 제거, 정산 파일 비교, 특정 양식으로 변환
3. 현재 수작업 소요시간: 예) 매주 2시간, 매일 30분, 건당 5분
4. 필요한 결과물: 예) 다운로드할 CSV, 엑셀 리포트, 구글시트 자동 업데이트
5. 희망 마감일
6. 샘플 파일 공유 가능 여부

실제 데이터 공유가 어렵다면 이름, 전화번호, 금액 같은 민감한 값은 가린 샘플도 괜찮습니다.
입력 파일과 원하는 결과 파일 예시가 있으면 가장 정확하게 확인할 수 있습니다.
```

## 2. 샘플 확인 후 추가 질문

상태 전환:

```text
NEEDS_SAMPLE -> QUALIFIED 후보
```

본문:

```text
샘플 파일 확인했습니다.

견적을 확정하기 전에 아래 내용만 추가로 확인하고 싶습니다.

1. 이 작업을 얼마나 자주 반복하시나요? 예) 매일 / 매주 / 월 1회
2. 결과 파일에서 반드시 유지되어야 하는 열이나 양식이 있나요?
3. 예외 케이스가 있나요? 예) 특정 상품 제외, 빈 값은 유지, 같은 주문번호 처리 방식
4. 납품 형태는 어떤 방식이 좋으신가요?
   - 웹 도구
   - 실행 스크립트
   - 엑셀/CSV 결과 파일 생성
   - 구글시트 자동화

이 답변을 기준으로 작업 범위와 금액을 정리해 드리겠습니다.
```

## 3. 견적 안내

상태 전환:

```text
QUALIFIED -> QUOTED
```

본문:

```text
확인한 파일과 반복 작업 기준으로 아래 범위로 진행 가능해 보입니다.

작업 범위:
- 입력:
- 처리:
- 결과:
- 제외:

예상 금액:
- 원

예상 납기:
- 영업일 기준 일

수정 범위:
- 최초 합의한 입력/처리/결과 범위 안에서 1회 수정 포함
- 파일 구조나 결과 양식이 크게 바뀌는 경우는 추가 범위로 산정

진행 가능하시면 아래 내용을 회신해 주세요.

1. 위 작업 범위로 진행 동의
2. 사용할 샘플 파일 최종본
3. 결과 파일 예시 또는 완료 기준

확정 후 작업을 시작하겠습니다.
```

## 4. 진행 확정

상태 전환:

```text
QUOTED -> WON
```

본문:

```text
진행 확인했습니다.

작업 범위는 아래 기준으로 잡겠습니다.

- 입력:
- 처리:
- 결과:
- 납기:
- 수정:

작업 중 파일 구조상 추가 확인이 필요한 부분이 있으면 먼저 문의드리겠습니다.
완료 후 결과물과 사용 방법을 함께 전달드리겠습니다.
```

## 5. 범위 초과 안내

상태 유지:

```text
QUALIFIED 또는 QUOTED 유지
```

본문:

```text
말씀 주신 추가 요청은 가능합니다.
다만 처음 확인한 범위보다 아래 항목이 늘어납니다.

추가되는 부분:
- 

영향:
- 작업 시간:
- 금액:
- 납기:

기존 범위로 먼저 진행할지, 추가 범위까지 포함해서 다시 견적을 잡을지 선택해 주세요.
```

## 6. 보류 또는 거절

상태 전환:

```text
NEW 또는 QUALIFIED -> LOST
```

본문:

```text
문의 주셔서 감사합니다.

현재 공유해 주신 조건만으로는 작업 범위나 완료 기준을 명확히 잡기 어렵습니다.
특히 아래 항목이 확인되어야 견적과 납기를 안내드릴 수 있습니다.

- 입력 파일 예시
- 원하는 결과물 예시
- 반복 작업 규칙

이 자료가 준비되면 다시 확인해 드릴 수 있습니다.
```

## 사용 규칙

- 확정 견적은 샘플 파일 확인 전에는 보내지 않는다.
- 개인정보가 포함된 파일은 가림 처리 요청을 먼저 한다.
- 결제나 계정 권한이 필요한 작업은 별도 범위로 분리한다.
- `docs/ops/pricing-guide.md`의 패키지 기준과 충돌하지 않게 쓴다.
- 답장을 보내기 전에는 사용자 승인을 받는다.
